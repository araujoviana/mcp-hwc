# HWC Pricing Tools — Design Spec

## Problem

The Huawei Cloud price calculator web UI is frustrating to use. Users need a way to get pricing/quotation for cloud resources through conversational AI, with the ability to persist, share, and export quotes.

## Approach

Extend the existing `mcp-hwc` MCP server with pricing tools. Use the BSS SDK as the primary pricing backend, with Playwright scraping of the HWC calculator as a fallback for services not covered by the SDK.

## Architecture

New module under `mcp_hwc/pricing/`:

```
mcp_hwc/
  pricing/
    __init__.py
    catalog.py        # Friendly-name -> HWC code mapping
    bss_pricing.py    # BSS SDK backend (primary)
    web_pricing.py    # Playwright scraping backend (fallback)
    models.py         # Data models (QuoteItem, QuoteResult, etc.)
    tools.py          # MCP tool definitions
    persistence.py    # Quote save/load/share
```

New dependency: `huaweicloudsdkbss>=3.1.196` (already added).

## MCP Tools

### `price_quote`

Primary quotation tool. Accepts a list of resources with partial or full specs.

**Input:**
- `resources`: list of resource descriptors, each with:
  - `service`: service name (e.g. "ecs", "rds", "evs", "obs")
  - `spec`: resource spec code or friendly name (e.g. "c6.large.2", "mysql.ha.xlarge.4")
  - `region`: region code or alias (e.g. "sa-brazil-1", "sao paulo")
  - `period_type`: "on_demand" | "month" | "year"
  - `period_num`: number of periods (default 1)
  - `quantity`: number of instances (default 1)
- `region`: default region for all resources (overridable per-resource)

**Output:** `QuoteResult` containing:
- `quote_id`: UUID for persistence
- `items`: list of `QuoteItem` with service, spec, unit_price, total_price, period
- `total_monthly`: estimated monthly cost
- `total_annual`: estimated annual cost
- `currency`: always "USD"
- `created_at`: timestamp

### `price_discover`

Discover available resource types and specs for a service.

**Input:**
- `service`: service name
- `region`: region
- `keyword`: optional filter keyword

**Output:** List of available `resource_type` and `resource_spec` codes with descriptions.

### `price_share`

Generate a shareable URL for a quote on the HWC price calculator.

**Input:**
- `quote_id`: ID of a saved quote

**Output:**
- `share_url`: URL that opens the HWC calculator with the quote pre-filled
- `method`: "playwright" (generated by browser automation) or "direct" (constructed from params)

**Implementation strategy (priority order):**
1. Attempt to construct a direct URL with query params if the calculator supports it
2. Fall back to Playwright: open the calculator, fill in resources, click "Share", capture the generated URL
3. If both fail, return a deep link to the calculator service page (not pre-filled, but navigated to the right section)

**Note:** The Playwright share method requires the user to be logged into HWC in the browser session. This is the most reliable approach but depends on an active session.

### `price_export`

Export a saved quote in a specific format.

**Input:**
- `quote_id`: ID of the quote
- `format`: "json" | "csv" | "terraform"

**Output:** Formatted content string.

### `price_list_quotes`

List saved quotes.

**Input:**
- `limit`: max results (default 20)
- `service`: optional filter by service

**Output:** List of quote summaries (id, services, total, created_at).

### `price_get_quote`

Retrieve a specific saved quote.

**Input:**
- `quote_id`: quote ID

**Output:** Full `QuoteResult`.

### Dynamic mapping (resource_spec)

Resource specs (flavors, instance types) are discovered at runtime via `list_service_resources` and cached in memory with a TTL of 1 hour. The cache key is `(service, region)`.

## BSS Pricing Backend

`bss_pricing.py` implements the primary pricing path:

1. Resolve `cloud_service_type`/`resource_type`/`resource_spec` via catalog
2. For subscription pricing: call `list_rate_on_period_detail` with `PeriodProductInfo`
3. For on-demand pricing: call `list_on_demand_resource_ratings` with `DemandProductInfo`
4. Parse response into `QuoteResult`
5. If SDK returns error for unknown service_type or resource_spec → raise `PricingNotAvailable` to trigger Playwright fallback

### BSS API details

**Subscription pricing** (`list_rate_on_period_detail`):
- Request: `RateOnPeriodReq(project_id, product_infos: list[PeriodProductInfo])`
- `PeriodProductInfo`: id, cloud_service_type, resource_type, resource_spec, region, available_zone, resource_size, size_measure_id, period_type (2=month, 3=year), period_num, subscription_num

**On-demand pricing** (`list_on_demand_resource_ratings`):
- Request: `RateOnDemandReq(project_id, inquiry_precision, product_infos: list[DemandProductInfo])`
- `DemandProductInfo`: id, cloud_service_type, resource_type, resource_spec, region, available_zone, resource_size, size_measure_id, usage_factor, usage_value, usage_measure_id, subscription_num

## Playwright Fallback

`web_pricing.py` implements the fallback path:

1. Navigate to `https://www.huaweicloud.com/pricing.html` or service-specific calculator
2. Select the service category
3. Fill in region, spec, period
4. Read the price from the DOM
5. Return `QuoteResult` in the same format

This is slow (~10-20s per resource) but guarantees coverage for services not yet in the catalog or SDK.

## Quote Persistence

`persistence.py` manages quote storage:

- Storage location: `~/.config/mcp-hwc/quotes/`
- Format: JSON files named `{quote_id}.json`
- Each file contains the full `QuoteResult`
- Quotes are immutable once created (no update, only create/read/list)

## Output Formats

### Text (default)
Human-readable summary with per-resource breakdown and totals.

### JSON
Structured `QuoteResult` object.

### CSV
Flat table with columns: service, resource_type, spec, region, period_type, period_num, quantity, unit_price, total_price, currency.

### Terraform
Generates `main.tf` with resource blocks for the quoted resources, with price as a comment on each resource. Uses the existing `huaweicloud` Terraform provider resource types.

## Conversational Flow

The AI (via opencode) handles the user interaction:

1. User describes what they need in natural language
2. AI calls `price_discover` to find available specs if needed
3. AI calls `price_quote` with the resolved resources
4. AI presents the result to the user
5. If user wants to share: AI calls `price_share`
6. If user wants to export: AI calls `price_export`

The user never needs to know HWC internal codes like `hws.resource.type.ec2`.

## Implementation Order

1. `models.py` — data models
2. `catalog.py` — service/spec mapping
3. `bss_pricing.py` — BSS SDK backend
4. `persistence.py` — quote storage
5. `tools.py` — MCP tool definitions (price_quote, price_discover, price_export, price_list_quotes, price_get_quote)
6. `web_pricing.py` — Playwright fallback
7. `tools.py` update — add price_share tool
8. Integration tests

## Resolved Decisions

- **Currency:** Always USD. No multi-currency handling needed.
- **Discounts:** Not in scope. Return raw list prices only — no discount lookup or application.

## Open Questions

- **Share URL format:** Need to investigate whether the HWC calculator supports pre-filling via query params, or if Playwright is the only way to generate shareable URLs. This should be the first implementation task.
